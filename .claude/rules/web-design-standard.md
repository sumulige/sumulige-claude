# Web 设计标准

> 定义什么是好的设计，而非仅提供资源

## 优先级说明

| 标签 | 含义 | 处理方式 |
|------|------|----------|
| 🔴 CRITICAL | 必须遵守 | 违反将显著影响用户体验 |
| 🟠 HIGH | 应该遵守 | 设计审查必查 |
| 🟡 MEDIUM | 建议遵守 | 提升设计质量 |
| 🟢 LOW | 可选遵守 | 风格偏好 |

---

## 🟠 HIGH: 核心哲学 [DESIGN-001]

```
"Visual design is not about expressing yourself.
 The key thing is the information you're trying to get across."
                                        — Garry Tan (YC CEO)
```

**奥卡姆剃刀**：能删则删。文字、线条、边框——如果移除不影响含义，就移除它。

## 🔴 CRITICAL: 三大铁律 [DESIGN-002]

| 铁律 | 含义 | 检查方式 |
|------|------|----------|
| **对比 = 重要性** | 粗/大/色 = 重要 | 如果都粗，就都不粗 |
| **接近 = 相关性** | 靠近的被感知为一组 | 相关内容放一起 |
| **视觉层级** | 对比 + 接近 = 导航路标 | 眯眼测试：第一眼看哪？ |

## 🟠 HIGH: 设计决策流程 [DESIGN-003]

```
需求 → 信息层级 → 排版 → 颜色 → 间距 → 交互 → 验证
```

### Step 1: 信息层级 (🔴)

```
问自己：
1. 用户必须看到什么？（主标题）
2. 用户应该看到什么？（副标题、CTA）
3. 用户可能看到什么？（详情、辅助）
4. 什么可以删除？（装饰、冗余）
```

### Step 2: 排版系统 (🟠)

| 元素 | 规则 |
|------|------|
| **字号** | 使用模块化标度 (1.25 / 1.333 / 1.414) |
| **行高** | 正文 1.4-1.6，标题 1.1-1.3 |
| **行长** | 45-75 字符，max-width + padding |
| **单位** | rem/em，永不 px（除 1px 边框） |

```scss
// 模块化标度示例
$scale: 1.25;
$base: 1rem;

$text-sm: $base / $scale;      // 0.8rem
$text-base: $base;              // 1rem
$text-lg: $base * $scale;       // 1.25rem
$text-xl: $base * $scale * $scale; // 1.5625rem
```

### Step 3: 颜色系统

| 原则 | 说明 |
|------|------|
| **克制** | 2-3 主色 + 中性色 |
| **深灰非纯黑** | 正文用 #333 而非 #000 |
| **对比度** | WCAG AA 最低 4.5:1 |
| **语义化** | 成功/警告/错误 一致 |

### Step 4: 间距系统

```scss
// 8px 基础单位
$space-1: 0.25rem;  // 4px
$space-2: 0.5rem;   // 8px
$space-3: 0.75rem;  // 12px
$space-4: 1rem;     // 16px
$space-6: 1.5rem;   // 24px
$space-8: 2rem;     // 32px
$space-12: 3rem;    // 48px
$space-16: 4rem;    // 64px
```

**90% 情况**：网格 + padding/margin + 好标题 + 对比 = 足够

### Step 5: 交互状态

| 状态 | 必须有 |
|------|--------|
| **默认** | 基础样式 |
| **悬停** | hover 反馈 |
| **焦点** | focus-visible:ring |
| **激活** | active 按下感 |
| **禁用** | disabled 视觉弱化 |
| **加载** | 骨架屏 / 进度 |

## 反模式检查

| 反模式 | 信号 | 正确做法 |
|--------|------|----------|
| **装饰过度** | 渐变+阴影+圆角+动画 | 每个元素只用一种强调 |
| **AI 审美** | Inter + 蓝紫渐变 + 16px 圆角 | 建立独特品牌 |
| **信息过载** | 首屏塞满内容 | 留白是设计元素 |
| **一致性缺失** | 3 种按钮样式 | 组件库 + 设计令牌 |
| **忽视移动端** | 桌面优先 | 移动优先 |

## 组件标准

### 导航栏 (Navbar)

```
✅ 固定高度 (56-64px)
✅ Logo 左侧
✅ 主导航居中或右侧
✅ CTA 按钮突出
✅ 移动端汉堡菜单
```

### 英雄区 (Hero)

```
✅ 一句话价值主张
✅ 副标题解释
✅ 明确 CTA
✅ 留白充足
❌ 轮播图 (用户不看)
```

### 表单 (Form)

```
✅ 标签在输入框上方
✅ 实时验证
✅ 错误信息在字段下方
✅ 移除不必要字段
❌ 占位符替代标签
❌ 冒号 (:) 后缀
```

### 卡片 (Card)

```
✅ 清晰边界 (边框或阴影)
✅ 内容层级 (标题 > 描述 > 操作)
✅ 一致间距
❌ 过多操作按钮
```

## 审查输出格式

```
【眯眼测试】
第一眼看到：[元素]
期望第一眼：[元素] → ✅匹配 / ❌不匹配

【层级检查】
- 主层级：[是否清晰]
- 次层级：[是否区分]
- 视觉噪音：[可删除项]

【对比度】
- 正文：[比值] → ✅ / ❌
- 链接：[比值] → ✅ / ❌

【一致性】
- 间距：[8px 倍数?]
- 字号：[模块化标度?]
- 颜色：[系统化?]

【移动端】
- 触摸目标：≥44px?
- 行长：合理?
- 可读性：字号?
```
