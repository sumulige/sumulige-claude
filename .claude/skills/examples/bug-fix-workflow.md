# Bug Fix Workflow Example

> Bug 修复工作流示例

**版本**: 1.0.0
**作者**: AI
**标签**: [example, bugfix, workflow]
**难度**: 中级

---

## 概述

展示系统的 Bug 修复流程，从发现问题到验证修复。

## 适用场景

- 修复已发现的 Bug
- 调试未知问题
- 生产问题排查

## 触发关键词

```
fix bug, debug, troubleshoot, issue
```

## 使用方法

### 修复流程

```
1. 问题确认
   ├─ 重现问题
   ├─ 收集日志
   └─ 确定范围

   ↓
2. 根因分析
   ├─ 代码审查
   ├─ 日志分析
   └─ 调试验证

   ↓
3. 修复实现
   ├─ 编写修复代码
   ├─ 添加测试
   └─ 更新文档

   ↓
4. 验证上线
   ├─ 测试验证
   ├─ Code Review
   └─ 部署上线
```

### 示例对话

```
你: 用户报告登录后 session 会立即失效

AI (使用本技能):
1. 问题确认：检查日志，发现 session 配置错误
2. 根因分析：cookie domain 设置不正确
3. 修复实现：更新配置文件
4. 验证上线：测试通过，部署修复
```

## 输出格式

| 阶段 | 输出 |
|------|------|
| 问题确认 | Bug 描述、复现步骤 |
| 根因分析 | 问题原因、影响范围 |
| 修复实现 | 修复代码、测试用例 |
| 验证上线 | 测试报告、部署说明 |

## 注意事项

- 先定位问题再动手修复
- 添加回归测试防止复发
- 记录问题根因
